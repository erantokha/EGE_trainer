## 5. Сквозные инварианты процесса

- **`?v=` cache-busting:** каждая волна, трогающая импортируемые модули
  (`picker.js`, `header.js`, CSS), → `node tools/bump_build.mjs`, иначе браузер
  возьмёт старое из кеша.
- **Red-zone overlay:** `tokens.css` (общие токены), общий сайдбар/навигация,
  общий грид аккордеона — explicit approval оператора, узкий scope, усиленное
  evidence. Speed-first не по умолчанию.
- **Общий CSS шарится с teacher** (`body[data-home-variant]`): любая правка
  сайдбара/аккордеона проверяется и на `home_teacher.html`.
- **Не удалять фичи молча:** «Умная тренировка» прячется, но не удаляется;
  переезд — отдельная постановка.
- **Скриншоты, не sleep:** teacher-статистика грузится асинхронно — ждать
  `waitForFunction`, не таймер (память teacher-restyle).
- **TaskCreate/TaskUpdate** по пунктам каждой волны (правило волн от 3 шагов).

### Принцип общих модулей (для всех волн)
Если правка — это **общий** модуль (токены, разметка/CSS сайдбара, грид
аккордеона через `body[data-home-variant]`), делаем её **сразу для ученика и
учителя** в текущей волне (это «бесплатно» — слой общий). Если правка
**teacher-специфична** (его навигация, его контент/статы) — выносим в
**отдельную teacher-волну потом**, не смешивая.

## 7. Предлагаемая последовательность

`Волна 0 (токены+каркас)` → `Волна 1 (сайдбар)` → `Волна 2 (шапка+тулбар)` →
`Волна 3 (аккордеон)` → `Волна 4 (рельс)` → `Волна 5 (мобилка+приёмка)`.

Волны 0–2 — низкий/средний риск, дают видимый «премиум» быстро. Волна 3 —
ядро и главный риск (DOM `picker.js`). Волны 4–5 — доводка и адаптив.
