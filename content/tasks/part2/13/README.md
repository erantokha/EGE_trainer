# Задачи №13 части 2: контракт решений

Эта папка хранит постоянный контент задач №13 части 2 и контракт оформления
эталонных решений. Для новых правок действует правило: сначала фиксируем
требование в контракте, затем правим данные или генератор.

## Главные файлы

- `solution_contract.json` — единственный машинный источник правил оформления и
  математической подробности решений №13.
- `solution_contract.html` — человекочитаемая HTML-витрина того же контракта.
  Страница читает JSON, поэтому не должна расходиться с ним руками.
- `13.trig.factor.json`, `13.trig.quad.json`, `13.trig.group.json`,
  `13.trig.homog.json`, `13.trig.other.json`, `13.log.json`, `13.exp.json` —
  манифесты прототипов №13.
- `img/` — картинки, включая тригонометрические окружности для решений пункта б).

## Проверка контракта

После любой правки решений №13 запускать:

```bash
node tools/check_part2_13_solution_contract.mjs
```

Скрипт читает `solution_contract.json`, проверяет все семь манифестов и пишет
отчёты:

- `reports/w13_1_fix/solution_contract_audit.json`
- `reports/w13_1_fix/solution_contract_audit.html`

Блокирующие нарушения уровня `error` означают, что выкладывать данные нельзя.
`warning` требует ручного просмотра. `note` фиксирует допустимые технические
замечания или будущую нормализацию.

## Ручной просмотр

Единый preview всех прототипов:

```text
reports/w13_1_fix/all_part2_13_preview.html
```

В нём проверяем глазами, что все задачи оформлены аккуратно: шрифт, размер
формул, отступы, ответы, окружности, отсутствие внутренних скроллов у формул.

## Процесс изменения правил

1. Если появляется новое требование, сначала добавить его в
   `solution_contract.json`.
2. Проверить, что `solution_contract.html` показывает новое правило.
3. Если правило можно проверить автоматически, добавить или связать проверку в
   `tools/check_part2_13_solution_contract.mjs`.
4. Исправить данные, генератор или batch-скрипт.
5. Запустить `node tools/check_part2_13_solution_contract.mjs`.
6. Открыть общий preview и проверить результат глазами.

Не оставлять правила только в переписке, отчётном HTML или разовом batch-скрипте.
Если правило влияет на будущие задачи, оно должно попасть в
`solution_contract.json`.
