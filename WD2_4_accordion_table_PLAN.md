# WD.2.4 — Аккордеон → таблица с прогресс-барами (Claude Design, Вариант 2)

Дата: 2026-06-05 · Зона: RED-ZONE (рендер `picker.js` + центральный UI главной ученика).
Самая рисковая волна редизайна главной. Не пушим — локальный показ + итерации.

## 1. Цель (target)
Строка раздела/подтемы как таблица:
- слева — стрелка-раскрытие (▶) + номер + название (+ 📊 «уникальные» у раздела), **под названием — прогресс-бар точности + %**;
- колонка «Покрытие» — числом X/Y;
- колонка «Задачи» — степпер «− N +»;
- шапка колонок «Тема и точность / Покрытие / Задачи»; лёгкая зебра; подтемы с отступом и тоньше.

## 2. Текущая структура (факт из picker.js)
- `.node.section .row`: `.countbox`(− N +) · `.home-section-badges`{`.home-section-pct`%, `.home-section-cov`X/Y} · `.section-title` · `.unique-btn` · `.spacer`. Плоский flex-ряд.
- `.node.topic .row`: `.countbox` · `.home-topic-badge`(%) · `.title` · `.spacer`.
- Шапка `.home-badges-head .row`: countbox-head · Процент/Покрытие · spacer.
- Рендер: `renderSectionNode` / `renderTopicRow` / `renderSectionBadgesHead` (picker.js).
- Бейджи пишет `picker_stats.js` (`setHomeBadge`: класс-цвет + `<b>%</b>` + `.small` + data-tip).

## 3. Главная сложность
Дизайн — 2D-сетка (бар ПОД названием; покрытие/степпер — колонками), а DOM плоский (всё в один ряд).
Название и 📊 должны быть в одной строке (flex), бар — под ними, покрытие — отдельной колонкой.

## 4. Подход (рекомендация): CSS-grid + хук `--pct` + минимальная безопасная обёртка
1. **picker_stats.js** — в `setHomeBadge` дополнительно класть `--pct` (0..100; «нет данных» → пусто/серый) на
   бейдж. Аддитивно (как WD.2.1). Класс-цвет/`<b>`/`.small`/data-tip — без изменений → charnet цел.
2. **picker.js** (`renderSectionNode`/`renderTopicRow`/`renderSectionBadgesHead`) — минимальная обёртка ряда под
   сетку: «строку названия» (стрелка+номер+название+📊) в контейнер, бейдж-бар — слот под ней.
   **Все классы/id/обработчики и `.section-title`/`.title`/`.countbox`/бейджи сохраняются** (charnet ищет их
   `querySelector`-ом по классу; обёртка их не прячет). Единственная правка рендера — узкая.
   - Альтернатива без правки рендера: `display:contents` + grid-areas. Но название+📊 ложатся хуже —
     рекомендую обёртку ради точности.
3. **home-student.css** — таблица: grid-колонки (название/бар | покрытие | задачи), бар (дорожка + заливка
   `--pct` + число %), покрытие числом, степпер `− N +`, шапка «Тема и точность / Покрытие / Задачи», зебра,
   стрелка ▶ (поворот на `.expanded`), отступ/тон подтем. Скоуп `body[data-home-variant="student"]`.

## 5. charnet / e2e
Снапшот секций/подтем читает: класс-цвет бейджа, текст `<b>`, `.small`, data-tip, текст названия
(`stats-snapshot.cjs:154-181`). Не меняю ни классы, ни текст, ни tip; `--pct`/обёртки не читаются →
**golden не дрейфует** (проверю прогоном). Интеракции (раскрытие/сворачивание, степпер ±, клик подтемы →
модалка прототипов) сохраняются — проверю.

## 6. Открытые решения
1. **Цвет бара:** синий по дизайну («не пёстро») ИЛИ красный/жёлтый/зелёный по точности (информативнее).
   Класс-цвет на бейдже остаётся в любом случае (charnet цел) — это только визуал бара.
2. **Дробить ли:** одной волной (рекомендую) или 2.4a (бары) + 2.4b (полная таблица).

## 7. Риск и митигейшн
Митигейшн: правка рендера — только обёртка (классы/обработчики целы); тяжёлое — на CSS; charnet зелёный;
полная локальная проверка (свёрнутый/раскрытый + интеракции + бары по реальным данным) тестовым учеником;
governance. Не пушим.

## 8. DoD
- Свёрнутый и раскрытый аккордеон = таблица Варианта 2 (бары, покрытие, степпер, зебра, шапка).
- Бары отражают реальную точность (`--pct`), цвет — по выбранному решению.
- Раскрытие/сворачивание, степпер (± меняет `#sum`), клик подтемы → модалка прототипов — работают.
- charnet зелёный (без правки golden или с осознанной регенерацией), governance зелёные.
- Скоуп только главной ученика; учитель/тренажёр не затронуты.
