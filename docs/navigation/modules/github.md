# Папка .github (L1)

Назначение

- автоматические проверки репозитория (CI) через GitHub Actions

Состав

- [.github/workflows/validate.yml](../../../.github/workflows/validate.yml)
  - запускается на push/PR
  - пытается выполнить валидацию контента (если в репозитории есть скрипт tools/validate-content.mjs)
  - публикует отчёт как артефакт workflow

Типовые задачи

- изменить правила валидации контента:
  - править сам workflow и/или добавить скрипт в tools
- временно отключить строгую проверку:
  - править validate.yml (например, сделать шаг non-blocking)

Тонкости/риски

- workflow может “тихо” пропускать валидацию, если скрипта нет (это нормально, но легко забыть)
- любые изменения в CI лучше тестировать на отдельной ветке, чтобы не блокировать релиз на Pages
