# Часть 2 (№13) — схема контента и спека конвертера

Контракт, которому следуют **и** конвертер (банк→тренажёр), **и** рантайм тренажёра. Контекст и
решения — `part2_integration_contract.md`. Текущий формат части 1 — `content/tasks/models/9.9.json`
(`topic → types → prototypes → {id, stem, answer_spec, answer}`); разведка — `reports/part2_recon/RECON.md`.

## Принцип
Не плодим новый формат — **расширяем существующий контракт задачи** опциональными полями. Все новые
поля живут **внутри тела задачи** (локальный JSON, грузится через `fetch` по `manifest_path`), поэтому
governance каталога (`tools/check_runtime_catalog_reads.mjs`) не нарушается. Каталог-метаданные
(тема→класс→метод→прототип→вопрос) — отдельный слой (см. W13.1), здесь не описываются.

## Схема прототипа части 2 (расширение)
```jsonc
{
  "id": "13.<класс>.<метод>.<источник>.<n>",   // напр. 13.trig.factor.46.1 (точную id-схему фиксирует W13.1)
  "part": 2,                    // маркер части 2 (часть 1 поле опускает)
  "class": "тригонометрическое",// класс уравнения (ось каталога, ур.2)
  "method": "вынесение общего множителя", // метод (ось каталога, ур.3; для лог/показ = пусто/лист)
  "uses": ["формулы приведения","формулы двойного угла"], // приёмы — ФАСЕТ (мульти-метка), данные на будущее
  "max_primary": 2,            // макс первичных за №13 (для scoring; часть 1 — свой)
  "stem": "а) Решите уравнение \\( … \\).<br>б) Найдите все корни …, принадлежащие отрезку \\( [..;..] \\).",
  "answer_spec": { "type": "manual" },   // НЕ автопроверка; проверка двухуровневая (ученик→учитель)
  "answer": {                   // эталонный ответ (для показа и сверки)
    "general": ["x = -\\tfrac{\\pi}{6} + \\pi n,\\ n\\in\\mathbb{Z}"],  // серии (LaTeX)
    "roots":   ["-\\tfrac{31\\pi}{6}", "-\\tfrac{25\\pi}{6}"]            // отбор корней (LaTeX)
  },
  "solution": {                 // ЭТАЛОН решения (показывается после попытки)
    "steps": ["3\\cos\\left(x-…\\right) - … = 0", "…", "\\operatorname{tg} x = …"], // пошаговая цепочка (LaTeX, по строке)
    "gen_groups": [ { "head": "\\cos x = 0", "series": ["x = …"] } ],  // общее решение столбиками (LaTeX)
    "below": ["x_{1} = -5\\pi - \\tfrac{\\pi}{6} = -\\tfrac{31\\pi}{6}"],// разложение корней по опорным (LaTeX)
    "figure": "img/part2/Б13-001.svg"   // окружность отбора — путь к SVG-файлу (через <img>)
  }
}
```
- **LaTeX** — те же делимитеры, что у части 1 (`\( … \)` / `\[ … \]`), рендерит MathJax тренажёра.
- **Окружность** — отдельный **SVG-файл** в `content/.../img/part2/`, показывается как `<img>` (Решение 5).
- `class` / `method` / `uses` дублируют то, что нужно каталогу (W13.1 решает, что из этого уходит в
  каталог-измерения, а что остаётся тегом тела).

## Конвертер банк → тренажёр (W13.0)
**Где:** новый скрипт в банке `~/Desktop/Вторая часть/13/_сборка/` (рядом с движком), напр.
`_export_trainer.py`. **Вход:** солверы `_solver_*.py` + пилотные (`solve_tg_tex`/`solve_factor_tex`/
`solve_quad_tex`) + `_card.circle_svg` + параметры всех 75 клонов (списки `CASES`/`CLONES` уже на уровне
модулей). **Выход:** в указанную папку репозитория тренажёра — `content/`-JSON по схеме выше + SVG-файлы
окружностей.

**Что делает (по каждому из 75 клонов):**
1. Получить от солвера структуру: `eqline_tex`, `chain` (без условия), `gen_groups`, `roots`, `ans_b`,
   интервал (¾ через `pick_interval_34`), и `root_decomp(roots)` → `below` + `refs`.
2. Сложить `stem` (а) уравнение + б) отрезок), `answer.general` (из gen_groups, с `n∈ℤ`), `answer.roots`.
3. `solution.steps` = `chain[1:]`; `solution.gen_groups`; `solution.below`; `solution.figure` =
   сохранённый `circle_svg(roots, A, B, refs)` в `img/part2/<cid>.svg`.
4. Проставить теги: `class` (Класс), `method` (полка по таблице раскладки), `uses` (Приёмы), `part:2`,
   `max_primary:2`, `answer_spec.type:"manual"`.
5. Сгруппировать клоны по (класс → метод → источник) — для каталог-метаданных W13.1 (формат уточнит W13.1).

**Верификация конвертера (DoD):** для всех 75 — JSON валиден и читается тем же лоадером, что часть 1;
SVG-файлы создаются; residual каждого `answer.roots` в исходный lhs < 1e-7 (переиспользовать проверку
из `_build_doc.py`); ни один клон без `class/method/uses`.

**Безопасно:** офлайн, пишет только в `content/` (новые файлы) — рантайм/схему/RLS не трогает. Это
первый исполняемый хэндофф (W13.0).
