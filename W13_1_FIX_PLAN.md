# W13.1-fix — Паритет части 2 по всем поверхностям (условие/названия/эталон)

Полировочная под-волна после W13.1. **НЕ red-zone** (чистый фронт-рендер: схему/каталог-данные/RPC/scoring
не трогаем — это уже сделано в W13.1 и относится к W13.2). Контекст и решения —
`docs/navigation/part2_integration_contract.md`; отчёт W13.1 — `reports/w13_1/W13_1_REPORT.md`.

## §1. Цель
W13.1 завёл часть 2 (№13) в **основные** поверхности — аккордеон + показ/эталон в **тренажёре**. Но
**вторичные** поверхности (уникальные прототипы, список, предпросмотр прототипа, модалка выбора
прототипов, рек-карточки) остались на «part-1»-рендере. Отсюда три дефекта для №13: (1) условие показывает
литеральный `<br>` вместо пунктов а/б; (2) везде видны технические слаги `13.trig.factor`/`13.log`;
(3) решение (эталон) недоступно нигде, кроме тренажёра — в т.ч. **в режиме листа и в уникальных
прототипах**, где открыть его важно. Цель — довести часть 2 до **паритета по всем экранам**.

## §2. Контекст
- Эталон/условие уже реализованы в `tasks/trainer.js` (хелперы `renderPart2Stem`,
  `buildPart2EtalonBlock`/`buildPart2EtalonContent`; весь DOM — `createElement`/`textContent`, без
  `innerHTML`, окружность через `<img>`). Их нужно **переиспользовать**, а не писать заново.
- Названия подтем в каталоге **уже человеческие** (`index.json`: «Вынесение общего множителя»,
  «Логарифмические» и т.д.). Слаги «текут» в UI там, где подпись строится из `id`, а не из `title`
  (модалка прототипов — `base_id`; unique — лейблы из id; шаблон `${id}. ${title}` в рек-карточках).
- `subtopic_id` (`13.trig.factor`) — **несущий** (каталог, unic, question_bank, фронт-группировка по
  префиксу класса). **Менять id нельзя** — правка только на уровне отображения.
- Порядок классов (как в аккордеоне): тригонометрические → логарифмические → показательные = **Тип 1/2/3**.
- Готча из W13.1: эталон-блок ломал grid карточки → оборачивался в область `.task-ans`. В списке/unique
  (много карточек) эталон = **сворачиваемый тоггл на карточку**, по умолчанию свёрнут, экранный.

## §3. Out of scope

- НЕ менять `subtopic_id`/каталог-данные/манифесты/`index.json`/SQL (id несущие; всё это закрыто в W13.1).
- НЕ менять рендер/поведение части 1 (№1..12) ни на одном экране.
- **Печать части 2 (бывший follow-up #3) — НЕ в этой волне** (отдельное продуктовое решение). Эталон-тоггл
  по умолчанию экранный; печать №13 (условие + место для решения, нужен ли эталон в печати) — за оператором.
