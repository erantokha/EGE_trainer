## 2. Что УЖЕ премиум на учителе (переиспользуем, НЕ переписываем)

Часть правил WD3 я делал **общими** (`body[data-home-variant]`, не только student),
поэтому учитель уже наполовину премиум:

- **Фон `--surface`** (WD3·в0) — общий.
- **Премиум-сайдбар** (WD3·в1): лого «Е», «МЕНЮ», карточка «До ЕГЭ», крупные
  иконки 30px, аватар. Teacher-пункты навигации («Мои ученики», «Выданные
  работы») уже на месте. Общий модуль.
- **Правый рельс премиум** (WD3·в4): `.start-card` (Подборка → Предпросмотр →
  Начать) + градусник `#scoreForecast` — 312px, radius 18, `--shadow-md`.
  Правила `.start-card` / `.score-forecast` / `.picker-right` не заскоуплены под
  student → действуют и на учителе.
- **Ряд аккордеона премиум** (WD2.4): синие полоски точности по `--pct`
  (color-mix accent↔slate), степпер справа, полоска под названием, зебра,
  ▶-треугольник — общий слой `home-student.css`.

**Вывод:** база (фон, сайдбар, рельс, ряд аккордеона) на учителе уже есть.
Перенос — это до-сборка дельты, а не «с нуля».

## 3. Дельта (что РЕАЛЬНО нужно сделать)

**A. Распространить общий премиум на teacher** (заскоуплено под `student`):
1. **Карта-обёртка аккордеона** (`.picker-left` → белая карта). У учителя
   аккордеон в `.ht-content-wrap`; до выбора ученика он пуст → нужна
   карта-пустое-состояние.
2. **Премиум-тулбар**: свитч «Перемешать» (`#shuffleToggle`), ghost-кнопки,
   фильтр — на teacher-тулбаре (`.ht-action-toolbar` + `#teacherFilterDropdown`).
3. **Пилюля «слабая тема»**: расширить scope на teacher (логика `.weak` в
   `picker_stats.setHomeBadge` уже ставит класс на обеих).

**B. Teacher-специфика** (адаптация, мокапа нет):
4. **Комбо выбора ученика → премиум-поле поиска** (`.ht-combo`/`.ht-combo-input`):
   иконка поиска, border/radius/фон под токены, выпадающий список
   `.ht-combo-list` как премиум-меню (карточка, hover-подложки, score-пилюли
   учеников), кнопка профиля `#studentProfileBtn` — премиум.
5. **«Создать ДЗ»** (`#createHwBtn` в карточке подборки) → премиум-CTA
   (вторичное действие под «Начать»: ghost или мягкий акцент).
6. **Пустое состояние** (ученик не выбран): карта аккордеона показывает
   приглашение «Выберите ученика, чтобы подобрать задачи», рельс/градусник —
   нейтральны.

**C. Мобилка + приёмка.**

## 4. Волны

> Принцип (как в WD3): каждая волна — проверяемый baseline. Общий CSS шарится с
> учеником → **после каждой правки scope проверять, что ученик НЕ сломан**.
> После волны: `bump_build`, governance, скриншоты ОБЕИХ главных.

### Волна WD4.0 — Общий премиум на учителя *(низкий риск, в основном scope)*
**Файлы:** `tasks/trainer/pages/home-student.css` (scope-флипы), `tasks/home_teacher.layout.css` (teacher-тулбар).
- **Карта аккордеона**: распространить `.picker-left` card-wrap на teacher
  (учесть `.ht-content-wrap` и пустой аккордеон). Тулбар (`.ht-action-toolbar`)
  → шапка карты.
- **Свитч «Перемешать»**: `body[data-home-variant="student"] #shuffleToggle` →
  `body[data-home-variant]` (оба). Проверить teacher-тулбар-layout.
- **Ghost-кнопки** «Выбрать все/Сбросить все» + фильтр `#teacherFilterDropdown`
  — премиум (как у ученика).
- **Пилюля «слабая тема»**: scope-флип на `body[data-home-variant]` (+ topic).
- **DoD:** teacher-аккордеон в карте; премиум-тулбар со свитчем; пилюля; ученик
  не сломан (скриншоты обоих).
- **Риск:** 🟧 общий CSS — scope-флипы могут зацепить ученика. Mitigation:
  точечные селекторы, скриншот-сверка ученика после каждого флипа.

### Волна WD4.1 — Премиум-поле поиска ученика *(teacher-специфика)*
**Файлы:** `home_teacher.html` (разметка комбо — минимально), `home_teacher.layout.css` (стили комбо + список).
- `.ht-combo` → премиум-поле: иконка поиска слева, border 1px/radius, фон
  `--panel`, focus-ring; placeholder «Поиск ученика…».
- `.ht-combo-list` → премиум-меню: карточка (border/radius/shadow), пункты с
  hover-подложкой, имя + score-пилюля (вторичный балл) ученика.
- `#studentProfileBtn` (иконка профиля) + `#studentComboClear` (×) — премиум.
- Шапка `.ht-head-row`: H1 «Выберите ученика» + поле в ряд, выверенные отступы.
- **DoD:** поле поиска и выпадающий список читаются как премиум; поведение
  (поиск/выбор/сброс) не тронуто (JS-контроллер комбо оставляем).
- **Риск:** 🟧 у комбо свой инлайн-JS (рендер списка) — меняем только вид (CSS),
  не структуру/id.

### Волна WD4.2 — «Создать ДЗ» + пустое состояние

## 5. Сквозные инварианты

- **Общий CSS шарится с учеником.** Каждый scope-флип (`student` → оба) → сверка
  скриншотом, что ученик не изменился негативно.
- **Teacher-специфику не ломаем:** комбо-JS, `teacher-student-view`,
  `--ht-thermo-h`, поток ДЗ (`hw_create.js`) — только вид, не логика.
- **`?v=` cache-busting** при правке импортируемых модулей → `bump_build`.
- **Red-zone:** общий слой `home-student.css`, teacher-навигация/комбо —
  узкий scope, усиленное evidence.
- **TaskCreate/TaskUpdate** по пунктам каждой волны.
