**RED-ZONE** (каталог-схема/RPC, `picker.js`). Контракт и решения —
`docs/navigation/part2_integration_contract.md`; схема контента — `docs/navigation/part2_content_schema.md`;
разведка — `reports/part2_recon/RECON.md`. Вход: данные из **W13.0** (готовые JSON части 2 + SVG).

## §1. Цель
Сделать №13 **видимым и навигируемым** на главной (ученик и учитель): после №12 с отступом, как
**3-уровневый аккордеон** (№13 «Уравнение» → класс → метод), с режимом «уникальные прототипы»; по нажатию
— показ задачи с условием и **эталоном** (пошаговое решение + окружность). Класс и метод — **реальные
измерения каталога** (чтобы потом собиралась статистика по методу). Баллы/запись попытки — **НЕ здесь**
(это W13.2): задача пока показывается и решается «вхолостую» (эталон по кнопке), без зачёта.

## §2. Контекст
Каталог сейчас 2-уровневый (`renderSectionNode` L1 / `renderTopicRow` L2 в `picker.js`), данные —
backend-измерения `catalog_*_dim` через RPC `catalog_*_v1`, исходник `content/tasks/index.json`. №1..12
сортируются `sort_order`+`compareId()`. 3-уровневого моста L1→L2→L3 нет. `unic`-механизм
(`catalog_subtopic_unics_v1`, `#protoPickerModal`, `unique.html`) — готов. `picker.js` — активный трек W2.

## §3. Out of scope
- НЕ scoring / запись попытки / градусник / ДЗ — это W13.2.
- НЕ двухуровневая проверка (учительский балл).
- НЕ менять рендер/поведение части 1 (№1..12).
- НЕ рефакторить `picker.js` за рамками добавления 3-го уровня (трек W2 ведёт декомпозицию отдельно).

## §4. Затрагиваемые файлы (уточняется recon-шагом §5.2)
