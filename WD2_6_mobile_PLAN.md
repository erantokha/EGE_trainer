## 4. Ключевые трансформации (медиа: телефон ≤ ~560px)
1. **Одна колонка + переупорядочивание**: `picker-grid` → column; прогноз (gauge) — НАВЕРХ (над аккордеоном); старт-карточка → в липкую нижнюю панель (CSS order/позиционирование).
2. **Прогноз → горизонтальная полоса**: reflow вертикального «градусника» (`.sf-track` верт.→гориз. бар; `.sf-fill` по ширине `--sec`; линия цели; вертикальные marks скрыть, показать «0/цель/100» снизу; «первичные/до цели» в строку). Данные те же — `picker_stats` НЕ трогаем. *Самая хитрая часть.*
3. **Липкая нижняя панель**: старт-карточка (`#previewBtn` + `#start`) → `position:fixed` снизу, в ряд: «Предпросмотр» (контур) слева + «Начать» (градиент, шире) справа; счётчик `#sum` → бейдж в «Начать». Blur-фон, padding под safe-area. Контенту — `padding-bottom`, чтобы не перекрывался.
4. **Аккордеон-строки мобайл**: степпер на верхнюю строку (рядом с названием), бар+%+покрытие — нижней строкой (переопределить десктопный grid 3×2 на мобильную раскладку).
5. **Шапка мобайл**: H1 компактнее; пилюля — аватар(?)+колокольчик, имя скрыть.
6. **Модалка предпросмотра** (наша WD.2.5) на мобайле — как нижний лист: прижата к низу, grabber, скругление сверху, без «Начать» внутри.

## 5. Риски / нюансы
- Зависит от WD.2.x (накоплено, не запушено) — мобайл продолжаем поверх.
- CSS-тяжёлая волна; самые хрупкие места — reflow прогноза в горизонталь, липкая нижняя панель, reorder gauge/start.
- picker.js/picker_stats/HTML-структуру по возможности **не трогаем** → charnet не затрагивается. (Если для горизонтального прогноза/панели понадобится минимальная правка DOM — отмечу отдельно.)
- Проверка на узком вьюпорте (Playwright 390–402px) тестовым учеником; charnet + governance зелёные.

## 6. Открытые решения (на подтверждение)
1. **Прогноз на мобайле:** reflow вертикального градусника в горизонтальную полосу **через CSS** (без правок DOM/JS) — пробую первым; если ляжет плохо — запасной вариант: отдельный мобильный strip-элемент (DOM + строка в `picker_stats`). Ок начать с CSS-reflow?
2. **Нижняя панель:** дизайн — **фиксированная** снизу (всегда видна). Беру `position:fixed` + отступ контенту. Ок?
3. **Модалка предпросмотра** на мобайле — наша WD.2.5 (конкретные задачи) в виде нижнего листа, **без «Начать» внутри** (как для десктопа). Ок?
4. **Пилюля:** аватар-кружок с инициалом мы откладывали (нет в header.js). На мобайле — показать имя+колокольчик без аватара, или всё-таки добавить аватар (тогда мелкая правка header.js, влияет и на десктоп)?
5. **Брейкпоинт:** телефон ≤ ~560px. Планшет (561–1024) — пока оставить как «узкий десктоп» (отдельно не оформляем в этой волне)?
