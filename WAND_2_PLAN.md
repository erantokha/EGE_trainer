# WAND_2_PLAN — волна 2: ученик целиком (ЭКОНОМ-РЕЖИМ)

Дата: 2026-06-12. Трек: WAND (`WAND_PLAN.md`). Базис: WAND.1 (`6e851315`).
Статус: утверждён оператором («Начинаю волну два в эконом-режиме»).

**Эконом-режим (решение оператора после анализа расхода токенов):**
вместо verifier-агента на каждую задачу — ДВА батч-verifier'а на волну
(оба на модели Sonnet): П-А после Д1–Д3 и П-Б финальный. Исполнитель
делает смоуки сам с минимумом скриншотов; живые тап-прогоны — только в
финальной приёмке; всё, что проверяется текстом (harness) — текстом.
Остальные инварианты процесса (план до реализации, stop-ask, autonomy
policy из `WAND_PLAN.md §7`, запреты на деструктив) — без изменений.

## 1. Цель

Полный функционал ученика: главная (прогноз, фильтры, аккордеон с
счётчиками, модалка прототипов, предпросмотр, черновик), тренировка с
проверкой ответов и записью попыток, отчёт с видео и аналогами,
статистика, ДЗ (список+бейдж, выполнение, архив). Эталон — финальный iOS
(Screens/Student, Screens/Homework, Screens/Stats, Screens/Shared) поверх
готового :core.

## 2–4. Контекст / Out of scope / Файлы

Контекст: :core (54/0) и дизайн-система готовы; табы-заглушки ученика
заменяются реальными экранами. Out of scope: всё учительское (WAND.3),
PDF и рисовалка (WAND.4), пошаговый режим (снесён оператором на iOS),
правки веб/SQL/iOS. Файлы: только `android/EGETrainerApp/app/**`
(screens/student, screens/homework, screens/stats, screens/shared,
правки RootNavigation/DevSupport), `reports/wand_2*`, этот план;
`:core` — только при дефекте (с юнитом и пометкой).

## 5. Чек-лист (TaskCreate Д1–Д6; задача закрыта = смоук исполнителя зелёный; батч-вердикты — П-А/П-Б)

- **Д1. Главная ученика** (порт StudentHomeView): карточка прогноза
  (термометр ScoreForecast, «+N до цели», поповеры «?»), фильтры-дропдаун
  (без/нерешённое/давно/нестабильно/слабые — бейджи состояний из
  pickingScreenSelf), ряд [фильтр][Выбрать все][Сбросить] равной высоты,
  «Перемешать», аккордеон 12 секций (счётчики −/N/+ на секцию и подтему,
  «Выбрать все» = +1 на КАЖДУЮ СЕКЦИЮ, прогресс-полоски, бейджи),
  карточка «Незавершённая тренировка» (draftStore), нижний бар
  [Предпросмотр | Начать (N)].
- **Д2. Модалка прототипов + предпросмотр** (порт ProtoPickerSheet,
  StudentPreviewSheet, PreviewQuestionCard): группировка по baseId,
  превью с формулами и картинками, степперы с капом, бейджи X/3 + дата
  (ProtoStatsCache, прогрев при раскрытии секции — без мигания), тап по
  строке подтемы открывает модалку, чип «+N»; предпросмотр без ответов,
  удаление, «Начать» учитывает удаления.
- **Д3. Тренировка** (порт TrainingRunView, TrainingReviewScreen,
  AnalogRunView, QuestionRunViews): список задач (StudentPickEngine.pick),
  ввод/«Проверить» (мгновенно, блокировка ответа), таймер, «Прервать» с
  черновиком (+ ответы), запись write_answer_events_v1 (source='test',
  meta.client='android'), отчёт (итоги, «Только неверные (N)», «Новая
  сессия», видео-плеер), «Решить аналог» (цепочка, meta.kind='hw_analog',
  исключения).
- **Д4. ДЗ** (порт MyHomeworksView, HomeworkRunView, HomeworkArchiveView):
  список со статусами/счётами, бейдж таба, выполнение по токену
  (get→start→ввод→подтверждение→submit_v2→результат с «только неверные»
  и аналогами), архив (offset 10, страница 50, «Загрузить ещё»).
- **Д5. Статистика** (порт StatsView): период 7/14/30/90, источник
  всё/ДЗ/тренировка, метрики last3/last10/период/всё время, покрытие,
  слабые подтемы.
- **Д6. Финальная приёмка**: e2e тап-сценарий ученика (подбор → 2-3
  задачи → проверка → прервать → продолжить → завершить → отчёт → аналог;
  сдача тестового ДЗ `6a43189f…` из WAND.0 через UI), запись в
  answer_events видна (harness analytics/прогноз сдвиг не требуется —
  достаточно успешного write), скриншоты ключевых экранов (light, +2
  dark), отчёт `reports/wand_2_report.md`.

## 6. Контракты
