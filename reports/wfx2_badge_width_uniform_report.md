**§5.6 — Доработка по фидбэку оператора: подтемный бейдж = ширина «%» раздела.**
После приёмки оператор уточнил, что уравнивание по `--home-topic-badge-w` дало
«суперузкие» бейджи. Изменено: `.home-topic-badge` width/min-width
`var(--home-topic-badge-w, auto)` → **`var(--home-pct-w, 40px)`** в обоих файлах
(base.css ~2592, home-student.css ~453) — теперь подтемный бейдж точности той же
фикс-ширины, что бейдж «%» раздела (`.home-section-pct` = `--home-pct-w` = 40px).
Обоснованно: подтемный бейдж рендерится через `setHomeBadge(el, p, 0, 0, …)` —
показывает ТОЛЬКО «%» (при count 0/0 `.small` пуст), контент идентичен section-«%»,
который уже живёт в 40px. Текст по центру — уже обеспечен базовым
`#accordion .badge{ justify-content:center }` (есть у обеих ролей: base.css 2405,
home-student.css 240); `margin-right:12px`/grid-позиция на внутреннее центрирование
не влияют, доп. правил не потребовалось. JS (`syncHomeTopicBadgesWidth` /
`--home-topic-badge-w`) не трогался — для ширины топик-бейджа он больше не
используется (no-op, вне scope). WFX2-исключение `:not(.home-topic-badge)` из
`width:auto`-сброса (§5.2/§5.3) оставлено — оно и нужно, чтобы правило ширины применилось.
Комментарии у затронутых правил обновлены.

### Обоснование `<N> = 96px` (§5.1)

96px = прежняя проверенная в проде `min-width` у `.modal-stats-badge` (до WFX-косметики
все proto-бейджи опирались на неё). Запас по контенту во всех состояниях:
- точность: «Не решал» (~74px с padding+border), «100% 3/3» (~72px), «33% 1/3»,
  нейтраль «— 0/0»; тултип «Загрузка…»/«Статистика недоступна» (текст бейджа = «—»);
- давность: «дд.мм.гг» напр. «21.02.26» (~74px — отсюда и была min-width:74px).
Самый широкий контент ≈ 74px < 96px → ни обрезки, ни переноса. `box-sizing:border-box`
гарантирует, что обе пилюли имеют РОВНО одинаковую внешнюю ширину.

## 6. На оператора (визуальная приёмка, §9 — обязательна для косметики)

Ширина — визуальный атрибут, не выражается в charnet-снимке, а добавлять e2e-спеки
вне scope (только §4-CSS). Деплой не нужен — смок локально против рабочего дерева
(`python3 -m http.server 8000`, `Cmd+Shift+R`):
- **Ученик:** главная → раскрыть несколько разделов → бейджи точности подтем одной
  ширины и **= ширине бейджа «%» раздела** (§5.6), «%» по центру пилюли; открыть
  подтему → модалка: точность и давность одной ширины, контент не обрезан («Не решал» /
  «100% 3/3» / дата).
- **Учитель:** выбрать ученика → то же по аккордеону (подтемный бейдж = section-«%») и модалке.
- Ресайз ≥1025px и уже — ширина остаётся единой, без обрезки.

Запасы по ширине: модалка `96px` (от проверенной min-width); подтемный бейдж `40px`
= `--home-pct-w`, той же ширины уже рендерится section-«%» с тем же «%»-контентом →
обрезки не ожидается. Если где-то покажется тесно — значение поднять в одной строке.
