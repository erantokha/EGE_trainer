# my_homeworks — аудит и план рестайла

Базовый визуальный язык уже близок к эталону (общие токены, `.panel`, скруглённые карточки, badge-пилюли), но страница живёт в «старом» layout-контуре: нет сайдбар-рельса/гамбургера, нет eyebrow+H1-связки, шапка собрана из header.js-кнопок («Антон / На главную»), а не из навигации эталона. Средняя дистанция от эталона: каркас другой, контент-карточки — почти ок.

## Десктоп
- Проблемы:
  - Нет левого сайдбар-рельса (56px, `body[data-home-variant]` + `.ht-sidebar*`), который есть на главных. Навигация вынесена в правый верх как кнопки `На главную` + пилюля `Антон` (header.js `.page-head-right` / `#userMenuWrap`) — на эталоне это спрятано (`display:none`), а навигация в рельсе.
  - Нет eyebrow-надстрочника над заголовком. Эталон: `ПОДГОТОВКА К ЕГЭ ПО ПРОФИЛЬНОЙ МАТЕМАТИКЕ` (eyebrow) + крупный H1 `Выберите темы…`. Здесь только `Мои ДЗ` H1 + рядом серая шестерёнка `⚙` (`.profile-gear-btn`), визуально похожая на мусорную иконку.
  - Вся страница завёрнута в одну большую внешнюю `.panel` (рамка + тень вокруг всего экрана). На эталоне внешней обёртки-панели нет — контент дышит на фоне `--bg`, панелями оформлены только смысловые блоки (карточки справа). Двойная вложенность «панель в панели» (внешняя `.panel` + `.myhw-card`) даёт «коробку в коробке».
  - Счётчики `Несданные: 4 / Всего: 14` (`.myhw-counters .muted`) — мелкий серый текст без структуры/иерархии, не в стиле stat-блоков эталона.
  - Контент центрируется в `.container` (`max-width:1080px; margin:28px auto`) без сдвига на рельс — после добавления рельса нужно учесть clearance, иначе наезд.
- План:
  - Добавить `data-home-variant="student"` на `<body>` (или эквивалент app-страницы) и подключить общий sidebar-DOM/JS как на главных. Источник рельса — `tasks/trainer/pages/home-student.css` (блок `WSF-student-sidebar`, `.ht-sidebar-panel` 56px→220px hover-expand) + инициализатор из `picker.js` (там строится `.ht-sidebar`, `#htSidebarOpen`, пункты навигации). Текущий пункт «Мои работы» (`#htNavWorks`) подсветить как активный.
  - Через `body[data-home-variant]` автоматически скроется правый блок `На главную`/`Антон` (`.page-head-right`/`#userMenuWrap`), а имя/выход уедут в футер рельса — то же поведение, что на эталоне.
  - Снять внешнюю обёртку `.panel` вокруг всего экрана: оставить `.container` как layout-врапер, а eyebrow+H1 и список карточек рендерить прямо на фоне `--bg` (как эталон). Тень/рамку оставить только у смысловых блоков.
  - Ввести eyebrow+H1: добавить надстрочник (тот же класс, что на главной — `.page-eyebrow` из home-student.css) над `<h1 id="pageTitle">Мои ДЗ</h1>`, увеличить кегль H1 до эталонного. Шестерёнку `⚙` либо убрать из заголовка, либо переоформить в аккуратную icon-btn справа в ряд заголовка (не «грязный» серый кружок).
  - Счётчики `Несданные/Всего` причесать в компактный stat-ряд (типографика как `.muted` эталона; при желании — мини-чипы), убрать инлайн-`style` из `my_homeworks.html` (счётчики обёрнуты в инлайновый flex) в `pages/my-homeworks.css`.
  - Добавить рельс-clearance: для `body[data-home-variant="student"]` уже есть `padding-left:56px` (→220px при `.ht-sidebar-open`) в home-student.css — убедиться, что `.container` центрируется в оставшейся ширине без наезда.

## Мобилка
- Проблемы:
  - Вместо гамбургера справа вверху — синяя круглая иконка-дом (`.home-icon-btn` / `#homeBtn` из header.js) + пилюля `Антон 🔔`. На эталоне справа вверху именно гамбургер (`.ht-burger` / `#htSidebarOpen`), открывающий drawer-сайдбар; иконки-дома нет.
  - Нет мобильного drawer-меню (`.ht-sidebar` open-state). Навигация недоступна в едином стиле — только «домой».
  - Шестерёнка `⚙` рядом с H1 на мобилке смотрится как случайный артефакт; нет eyebrow.
  - Горизонтального скролла на скриншоте не видно (хорошо), но это надо удержать после добавления drawer/overlay — `.ht-sidebar-overlay` и фиксированная панель не должны давать overflow-x.
  - Карточки `.myhw-card` сами по себе ок (badge `Сдано`/`Не сдано` читаемы), но они снова вложены во внешнюю `.panel` → лишняя рамка по краям экрана, съедает горизонтальные поля.
- План:
  - Тот же `data-home-variant="student"` + sidebar-DOM → на мобилке автоматически включится `.ht-burger` (в home-student.css `#htSidebarOpen` скрыт только `@media (min-width:1025px)`, на мобилке `display:flex`) и drawer-overlay как на эталоне. Гамбургер позиционировать справа вверху (mobile-правила home-student.css).
  - Убрать/спрятать `#homeBtn` (`.home-icon-btn`) и правую auth-пилюлю через тот же `body[data-home-variant]`-гейт, что и на десктопе (домой — пункт в drawer).
  - Снять внешнюю `.panel`-обёртку → карточки на чистом фоне с мобильными полями эталона (`body:not([data-home-variant]) .container{padding:0 10px}` сейчас не сработает после добавления `data-home-variant` — задать мобильные поля контейнера в варианте student, как на главной).
  - Добавить eyebrow+H1 в мобильном масштабе (как `.page-eyebrow` эталона), шестерёнку убрать/переоформить.
  - После изменений проверить: нет горизонтального скролла, drawer открывается/закрывается, iOS без зума (viewport уже `width=device-width,initial-scale=1` — ок).

## Риск/функционал
- JS страницы: `tasks/my_homeworks.js` (рендер списка ДЗ через RPC), `app/ui/header.js` (initHeader — имя/меню/«домой»), `theme.js`, меню-кнопка «Архив работ» (`#myHwMenuBtn`/`#myHwMenu` → `my_homeworks_archive.html`). При перестройке шапки НЕ потерять: контейнеры рендера `#myHwList`, `#myHwEmpty`, `#myHwPending`, `#myHwTotal`, `#myHwStatus`, `#pageTitle`, а также обработчик меню/архива (`#myHwMenuArchive`).
- Red-zone: добавление `data-home-variant` + sidebar затрагивает общий навигационный/layout-каркас (рельс из `home-student.css`, инициализатор из `picker.js`) и поведение `header.js` (скрытие `.page-head-right`). Это общий слой нескольких страниц — менять узким scope, с явным operator approval, не ломая главные ученика/учителя. Логику auth/RPC (`my_homeworks.js`, header user-menu, выход) не трогать — только обёртка/навигация/CSS.
- `?v=` cache-busting: правки `pages/my-homeworks.css` и `my_homeworks.html`/JS требуют `node tools/bump_build.mjs` (страница тянет CSS/JS с `?v=2026-06-07-88`).
- Карточки ДЗ (`.myhw-card`, badge ok/warn, клик по карточке) визуально уже близки к эталону — не переусложнять, основной фокус на каркасе (рельс/гамбургер, eyebrow+H1, снятие внешней панели).
