# my_students — аудит и план рестайла

Базовый визуальный язык близок к эталону (общий `tokens.css`/`base.css`, карточки `.panel`/`.stat-mini`, те же радиусы/тени/типографика), НО страница **не подключена к навигационному каркасу учителя**: нет сайдбар-рельса на десктопе, нет гамбургера на мобилке, нет связки `eyebrow + крупный H1`, шапка осталась дефолтной (header.js: «Антон» + «На главную»). По «визуальному языку карточек» — близко; по «каркасу страницы» (рельс/бургер/шапка) — далеко. Severity: medium.

## Десктоп

- Проблемы:
  - **Нет левого сайдбар-рельса.** Эталон `home_teacher` рендерит фиксированный рельс 56px (`#htSidebar` / `.ht-sidebar-panel`, hover/open → 220px) с пунктами «Мои ученики / Выданные работы / Профиль» + юзер/выход в подвале. На `my_students.html` рельса нет вовсе — у `<body>` отсутствует `data-home-variant`, не подключён `tasks/trainer/pages/home-student.css` (где живут все `.ht-sidebar*` / `.ht-burger`), нет блока `#htSidebar` и его inline-обвязки. Это главное расхождение каркаса.
  - **Шапка дефолтная вместо «eyebrow + H1».** Эталон: `.home-eyebrow` («ПОДГОТОВКА К ЕГЭ…») над крупным `.home-h1`. Здесь — мелкий `<h1 id="pageTitle">Мои ученики</h1>` (16-px паттерн `page-head`), а справа торчат пилюля «Антон» и «На главную» из `header.js` (на эталоне `.page-head-right`/`#userMenuBtn` спрятаны через `body[data-home-variant]`, навигация уходит в рельс).
  - **Контент в карточке-`.panel` вместо full-width под рельс.** Эталон делает `#picker > div` прозрачным (без карты/тени/радиуса) и тянет контент в область справа от рельса (`main { margin-left:56px }`). Здесь весь контент завёрнут в одиночную белую `.panel` по центру без учёта рельса — другой композиционный приём.
- План:
  - Добавить `data-home-variant="teacher"` на `<body>` и подключить `tasks/trainer/pages/home-student.css` (после `base.css`) — это включит общие стили `.ht-sidebar*`, `.ht-burger`, `.home-eyebrow/.home-h1`, скрытие `.page-head-right`.
  - Перенести/скопировать из `home_teacher.html` блок `#htSidebar` (nav: `#htNavStudents`/`#htNavWorks`/`#htNavProfile`, foot: `#htSidebarUserBtn`/`#htSidebarLogout`) и его inline-JS-обвязку (open/close/overlay/nav-переходы, ~строки 779–870 эталона). Лучше вынести этот сайдбар-инициализатор в общий модуль (напр. `app/ui/teacher_sidebar.js`) и переиспользовать на обеих страницах, чтобы не плодить копипасту.
  - Переписать шапку под эталон: обернуть заголовок в `.home-head-text` с `.home-eyebrow` + `.home-h1`, добавить кнопку-гамбургер `.ht-burger.ht-burger-open#htSidebarOpen` в `#appHeader` (на десктопе её скроет `@media(min-width:1025px) #htSidebarOpen{display:none}`).
  - Снять карточный вид с обёртки списка под рельс: либо повторить teacher-приём (`#picker > div` прозрачный + `main margin-left:56px`), либо оставить `.panel`, но добавить desktop-клиренс под рельс 56px, чтобы карточка не наезжала на рельс. Выбрать один подход и согласовать с эталоном (предпочтительно как у `home_teacher`).
  - Панель фильтров (поиск/«Добавить»/«Проблемные»/селекты дней и источника) перевести с инлайн-стилей на токены/утилиты как в эталоне (border `var(--border)`, radius 10–12px) — визуально уже близко, но инлайны лучше унифицировать.

## Мобилка

- Проблемы:
  - **Нет гамбургера и сайдбара.** Эталон на мобилке: гамбургер справа вверху (`.ht-burger#htSidebarOpen`), открывающий выезжающую панель `.ht-sidebar` с оверлеем. Здесь вместо него — аватар-кружок «Антон» + иконка «домой» из `header.js`; навигации по разделам учителя нет.
  - **Шапка не как у эталона.** Нет eyebrow, заголовок крупный, но без верхнего eyebrow-строки; справа — дефолтные элементы header.js, а не гамбургер.
  - Горизонтального скролла/зума не замечено: карточки одной колонкой (`@media(max-width:560px) .students-grid{1fr}`), `.stat-mini` 2-в-ряд, переполнения нет — это ок и менять не нужно.
- План:
  - После подключения `data-home-variant="teacher"` + `home-student.css` + блока `#htSidebar` мобильная навигация заработает автоматически (гамбургер виден, `.page-head-right` скрыт, панель выезжает по `.open`).
  - Убедиться, что гамбургер позиционируется в правом верхнем углу `#appHeader` (эталон: `#appHeader{position:relative}` + mobile-правила позиционируют `.ht-burger` абсолютно справа). Для my_students нужно добавить аналогичное правило позиционирования бургера в шапке (в эталоне оно teacher-специфично в `home_teacher.layout.css`).
  - Сохранить текущую одноколоночную раскладку карточек и `viewport=width=device-width,initial-scale=1` — горизонтального скролла нет, iOS-зум не провоцируется; ничего ломать не надо.

## Риск/функционал

- **RPC (red-zone, не ломать):** страница вызывает `list_my_students`, `teacher_students_summary`, `add_student_by_email`, `remove_student` через `supabase-rest.js` (см. `tasks/my_students.js`). Рестайл — чисто разметка/CSS, RPC-контрактов не касаться.
- **Routing/click-логика:** клик по `.student-card` ведёт на `student.html` (формирование URL в my_students.js ~стр. 341–357). Перенос карточек/обёртки не должен снять обработчик `card.addEventListener('click', …)` и id-узлы `#studentsList`, `#searchStudents`, `#addStudentInlineBtn`, `#filterProblems`, `#summaryDays`, `#summarySource`, `#addStatus`, `#pageStatus` — все они читаются JS, переименовывать/удалять нельзя.
- **header.js / auth (red-zone):** инициализация `initHeader({isHome:false})` нужна (имя пользователя для подвала сайдбара, скрытый `#menuLogout`/`#userMenuBtn` используются эталонной обвязкой сайдбара для выхода). При скрытии `.page-head-right` через CSS оставлять элементы в DOM, не удалять.
- **Сайдбар-JS:** если выносить инициализацию сайдбара в общий модуль — не дублировать id (`#htSidebar*`, `#htNav*`) и убедиться, что навигационные переходы (Мои ученики/Выданные работы/Профиль) ведут на корректные URL для роли teacher.
- **CSS cache-busting:** добавление нового `<link>` на `home-student.css` и правки шаблонов требуют `node tools/bump_build.mjs` (импорт с `?v=`), иначе кэш подтянет старое. Общий `base.css`/`tokens.css` — общий каркас (осторожно с побочными эффектами на другие страницы; здесь правки локальны к my_students-разметке).
