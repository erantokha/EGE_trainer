# hw_create — аудит и план рестайла

Страница в «старом» визуальном языке: одинокая `.panel` по центру с шапкой `.page-head` (H1 + кнопки в ряд), без eyebrow, без сайдбар-рельса и без гамбургера. От эталона (home_teacher) далеко: нет фирменной навигации-каркаса, типографика и раскладка не совпадают. Контент (поля, чекбоксы, dashed-«editable» кнопки) выглядит как черновой, а не как карточки эталона.

## Десктоп
- Проблемы:
  - Нет левого сайдбар-рельса (`.ht-sidebar-panel` / `.ht-sidebar-nav`), который есть на home_teacher как постоянная навигация. Страница висит одной центральной панелью.
  - Нет шапки-эталона: отсутствует `.home-eyebrow` («Подготовка к ЕГЭ по профильной математике») над крупным `.home-h1`. Сейчас просто `<h1 id="pageTitle">Создание ДЗ</h1>` внутри `.page-head`.
  - В `.page-head` справа в ряд стоят кнопка «Печать», «Антон» (user-menu) и «На главную» — на эталоне навигация и user-меню живут в рельсе/футере рельса, а не в шапке контента. Кнопка «На главную» вообще лишняя при наличии рельса.
  - `<body>` без `data-home-variant="teacher"`, поэтому не применяется общий студенческий/учительский CSS-вариант (отступы, фон, токены рельса).
  - Подключён только `pages/hw-create.css`; нет `home-student.css` + `home_teacher.*.css`, в которых лежат правила рельса/eyebrow/H1/burger (≈50 правил `.ht-sidebar*` в home-student.css).
  - Внутренние контролы не в стиле эталона: dashed-кнопки `.editable-btn` («ДЗ 07.06», «Описание»), нативный `<select>` без класса карточки, inline-`style` вместо классов — разнобой радиусов/границ/паддингов против карточек эталона.
  - Огромное пустое поле под панелью: контент не дотянут до сетки/высоты, как у эталона с правой колонкой.

- План:
  - Добавить `data-home-variant="teacher"` на `<body>` и подключить тот же набор стилей, что у home_teacher: `tokens.css`, `base.css`, `home_teacher.mobile.css`, `home_teacher.layout.css`, `home-student.css`, `home_student.mobile.css` (плюс оставить `pages/hw-create.css` для специфики формы). Синхронизировать `?v=` через `node tools/bump_build.mjs`.
  - Перенести разметку рельса из home_teacher: блок `<nav class="ht-sidebar-panel">` (head с `.ht-sidebar-close`, `.ht-sidebar-nav` с пунктами «Мои ученики» / «Выданные работы» / «Профиль», `.ht-sidebar-foot` с user-btn + logout) и обёртку-шелл, дающую двухколоночную раскладку рельс+контент. Подвесить навигацию рельса на существующие переходы (см. `htNav*` обработчики в home_teacher.html).
  - Перестроить шапку контента под эталон: `.home-eyebrow` + `.home-h1` («Создание ДЗ»). Кнопку «Печать» оставить как `data-header-extra` (header.js её и так раскладывает), theme-toggle оставить; user-menu и «На главную» убрать из шапки — они уходят в рельс. Кнопку «На главную» удалить (навигация — через рельс/логотип).
  - Привести внутренние контролы к карточному языку эталона: `.editable-btn` и `<select class="input">` оформить как поля/кнопки на токенах (`--border`, `--radius`, `--panel-2`), убрать inline-`style` в пользу классов; «Создать ссылку» сделать `btn primary` в едином стиле; чекбоксы — как в эталоне.
  - Убрать центрирование одинокой панели: контент-колонка в шелле должна занимать ширину рядом с рельсом, как на home_teacher.

## Мобилка
- Проблемы:
  - Нет гамбургера справа вверху (на эталоне — `#htSidebarOpen` / иконка `home_nav`), который открывает выезжающий рельс. Сейчас вверху справа кнопка «Антон» + иконка печати + домашняя иконка — это не паттерн эталона.
  - Нет eyebrow над H1; шапка — это `.page-head` со скученными кнопками, не мобильная шапка эталона.
  - Панель — узкая центрированная карточка с большими боковыми полями; у эталона контент тянется на ширину вьюпорта с правильными паддингами.
  - Контролы (select, dashed-кнопки) визуально мельче/иначе, чем карточки эталона; нет общей мобильной типографики.
  - Горизонтального скролла на скриншоте не видно, но из-за inline `min-width:240px` у `<select#assignStudent>` есть риск переполнения на узких экранах — проверить после переноса в шелл.

- План:
  - После добавления `data-home-variant` + мобильных CSS эталона мобильная шапка и гамбургер появятся из общего контура; добавить разметку открытия рельса (`#htSidebarOpen` с svg-иконкой) в шапку контента, как в home_teacher, и связать с `.ht-sidebar-panel` (выезжающая панель + backdrop, обработчики open/close из home_teacher.html).
  - Eyebrow + H1 на мобилке — те же `.home-eyebrow` / `.home-h1`, размеры берутся из `home_student.mobile.css`.
  - Убрать из мобильной шапки «На главную» и user-кнопку (user уходит в футер рельса), оставить печать/тему как `data-header-extra`.
  - Снять `min-width:240px` со `<select>` (заменить на `width:100%`/`max-width:100%`), чтобы исключить горизонтальный скролл; проверить, что модалка выбора задач (`#taskPickerModal`) на мобилке тоже без overflow и с dvh (см. память про iOS 100vh/dvh — модалка не должна «уезжать»).
  - Проверить отсутствие горизонтального скролла после переноса в шелл на 360–390px.

## Риск/функционал
- На странице живая логика создания ДЗ: `tasks/hw_create.js` импортирует `createHomework`, `createHomeworkLink`, `listMyStudents`, `assignHomeworkToStudent` (homework RPC-домен) и `supabase`/`getSession`/`signOut`/`finalizeOAuthRedirect` (auth lifecycle). Это red-zone (auth-flow + RPC): рестайл должен быть чисто разметочно-CSS, нельзя трогать ID/потоки, на которые завязан JS — `#createBtn`, `#assignStudent`, `#assignStatus`, `#status`, `#linkBox`/`#hwLink`/`#openLinkBtn`/`#linkMeta`, `#titleBtn`/`#titleInput`, `#descBtn`/`#descInput`, `#shuffle`, `#addTaskPlus`, `#toggleAdded`, `#addedBox`/`#fixedCards`, вся модалка `#taskPickerModal` (`#tpNav`/`#tpPath`/`#tpList`/`#tpSelectedCount`/`#tpAddSelected`/`#tpHint`/`#tpClose`).
- Шапка инициализируется через `app/ui/header.js` (`initHeader`) и `print_btn.js` (`initPrintBtn`); элементы с `data-header-extra="1"` раскладываются этим кодом — при переносе печати/темы сохранить атрибуты и `#printBtn`/`#themeToggle`, иначе сломается раскладка шапки и user-menu.
- При добавлении рельса переиспользовать обработчики навигации/логаута из home_teacher (htNav*/htSidebar*), не дублировать auth-логику; logout идёт через тот же `signOut`. Менять только разметку/классы/CSS, не SQL и не RPC-вызовы.
- Обязателен bump build id (`?v=`) после правки HTML/CSS — иначе кеш подтянет старую версию.
