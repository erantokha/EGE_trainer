## 2. Зафиксированные решения / находки

1. **Тренировка — full-screen поверх табов** (аналог fullScreenCover iOS):
   `activeRun` в StudentTabScaffold; тап «Начать»/«Продолжить» открывает
   TrainingRunScreen, закрытие возвращает на табы и обновляет бейдж ДЗ.
2. **testTagsAsResourceId** включён (MainActivity) — для скриптовой
   тап-приёмки через `uiautomator`/`adb input` по resource-id; обёрнут в
   `@ExperimentalComposeUiApi`.
3. **CountStepper/фильтр/«Выбрать все»** проверены вживую: +1 на двух
   подтемах → подборка собралась → «Начать» открыл тренировку с 2 задачами.
4. **Прогноз-паритет:** главная показала «первичные 2,88 / 16 из 100»;
   вторичный балл совпадает с harness `forecast.computed` (16). Небольшое
   расхождение первичного (2,86→2,88) — следствие записанной тестовой
   попытки между прогонами, не дефект формулы.
5. AlertDialog (подтверждение завершения) живёт в отдельном окне Compose —
   его testTag не виден uiautomator'у; для тапа по кнопке диалога в
   скриптах используются bounds (отмечено для будущих приёмок).
