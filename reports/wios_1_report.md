### Фикс-волна по ревью оператора (2026-06-11, 23:10–23:45)

Оператор отклонил сдачу: «Выбрать всё» = 84 задачи, предпросмотр «2 из 84 /
банк исчерпан», визуал ≠ мобильный веб, бейджи без дат у учителя и поздний
рендер, слабая рисовалка. Диагностика и фиксы:

1. **Диагностика resolve_batch против прода** (root-cause first): RPC исправен —
   96 topic-запросов → 84 задачи за 1,8 с; `global_all` и section-запросы
   работают. «2 из 84» = активный фильтр «Давно не решал» (stale) у выбранного
   ученика: для erantokha stale даёт ровно 2 кандидата. Ложный текст «банк
   исчерпан» был моей эвристикой.
2. **«Выбрать всё» учителя** — порт bulkPickAll: +1 на СЕКЦИЮ (12), не на
   подтему (84); у учителя появились section-бакеты (CHOICE_SECTIONS), они
   же прокинуты в создание ДЗ.
3. **Честный shortage** — из `shortages[]` ответа RPC: «Не хватило N задач:
   по фильтру „…" подходящих кандидатов больше нет (K тем). Снимите фильтр…»;
   активный фильтр показан чипом в шапке предпросмотра; потери сборки
   контента — отдельная строка. Скриншот: `15_preview_stale.png`.
4. **Даты у учителя в модалке** — корень: я звал `question_stats_for_teacher_v2`
   с несуществующим параметром `p_topic_id` → тихий 404. Сигнатура проверена
   против прода: только p_student_id + p_question_ids. Исправлено; даты
   рендерятся (`16_proto_teacher_dates.png`).
5. **Прогрев статистики (порт WFX1)** — `ProtoStatsCache` (TTL 60 с,
   дедупликация прогревов): прогрев при раскрытии секции у обеих ролей,
   модалка открывается с готовыми бейджами; вёрстка строк подтем поправлена
   (бейдж состояния — отдельной строкой).
6. **Визуальный паритет с web-reference**: фильтр → компактный дропдаун в
   строке с «Выбрать/Сбросить все» (обе главные), у учителя добавлен

7. **Рисовалка** — апгрейд на системную палитру PencilKit (PKToolPicker:

### Фикс-волна №2 по тестам оператора на устройствах (2026-06-12, ночь)

1. **Растянутые карточки** (текст → многоэкранный пропуск → рисунок) —
   регрессия моего LazyVStack: высота MathTextView мерилась один раз по
   scrollHeight viewport'а, а LazyVStack создаёт ячейку до назначения полной
   ширины (WebKit-тест: тот же текст при 20pt — 504px, при 350pt — 120px;
   повторного замера не было). Фикс: замер внутреннего контейнера `#c` +
   ResizeObserver — высота пересообщается при каждом изменении размеров.
2. **8–10 с подбор с фильтром у ученика** — не сервер (фильтрованный resolve
   0,9–1,5 с во всех формах), а моя последовательная реализация
   pickFiltered: 12 RPC по бакету = 11,3 с (замер) против 0,9 с одним
   батчем с тем же составом. Фикс: один батч со всеми бакетами
   (атрибуция scope_kind/scope_id из ответа, ротация поверх). Урок: аудит
   №1 мерил сценарий без фильтра — корень жил в фильтрованной ветке.
3. Обновление установлено на оба iPhone по воздуху (Wi-Fi, без кабеля).
