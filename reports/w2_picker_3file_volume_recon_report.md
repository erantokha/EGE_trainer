## 1. Целевые 3 файла

`picker.js` остаётся entry + **всё ролевое** (движок учителя НЕ трогаем). Из него вычищаются два чистых листа:

| Файл | Что внутри |
|---|---|
| **`picker.js`** (entry + roles) | движок added-tasks (1216), аккордеон, счётчики, прото-оболочка, модалки, last-10 ученика, smart, **оба рендерера статистики** (glue), boot, save-and-go, состояние module-level |
| **`picker_common.js`** | чистые утилиты, math, manifest/stem, resolve/preview-билдеры, esc/compareId |
| **`picker_stats.js`** | переписываемый stat-юнит: писатели бейджей, forecast/thermo, model-билдеры, rec-хелперы, loading/reset/width |

(Различие с полной архитектурой: stats как один лист вместо `stats_view`+`stats_model`; роли НЕ разделяются на teacher/student; движок и state остаются в `picker.js`.)

## 2. Метод

Программный подсчёт по `reports/w2_0_artifacts/function_inventory.csv` (176 функций с точными line-спанами).
Классификация: явные множества COMMON и STATS, всё остальное → roles(`picker.js`). Сумма строк по телам функций + top-level остаток.

## 4. Честный итог: что даёт и чего НЕ даёт

**Получаешь:** ✅ статистика изолирована в ~580-строчный юнит (главная цель — переписываемость); ✅ чистая переиспользуемая база `common`.

**НЕ получаешь:** ❌ широкой локальности — `picker.js` остаётся ~3950 строк (движок 1216 + остальное ролевое на месте). Чтобы тронуть аккордеон — всё ещё большой файл. Локальность «по фиче везде» требует дорогого teacher/student split (полный путь дизайн-отчёта).

## 5. Ключевая находка: 3-файловый путь обходит ОБА самых рисковых шага полного плана

Извлекаемые листья **чисты**: писатели берут `(element, value)`, model-билдеры берут `payload`/`SECTIONS` (передаётся параметром), утилиты не читают `CHOICE_*`. Следствия:

- **Своп shared-state (~90 ссылок) НЕ нужен** — изменяемое состояние остаётся module-level в `picker.js`, рендереры (читающие его) тоже там. Это снимает R2 полного плана.
- **Экстракция движка (1216) НЕ нужна** — снимает R1 (самый страшный) полного плана.
- **Нет цикла `state ↔ engine`** (adversarial-заметка §9 дизайн-отчёта) — потому что state не выносится.
