## Решения классификации (по контракту = SoT)

- **Метод-полка** — строго по таблице раскладки контракта (по источнику 13.NN), **не** из
  `_anno_final.json` (план §5.2/§7: «брать по таблице, не по догадке»). Правило «квадратное vs
  вынесение» уже зашито в распределение источников по полкам в контракте.
- **Класс** = `_anno_final.json.klass`, **кроме** override по контракту: `13.92 → показательное`,
  `13.93 → логарифмическое` (anno помечает их `смешанное` — показ./лог. оболочка; каталог-ось класса
  в контракте разводит их на отдельные листы). Все триг-источники в anno = `тригонометрическое`.
- **Приёмы (`uses`)** = `_anno_final.json.priemy` (словарь заморожен 2026-06-16,
  `_classification_spec.md`). У всех 75 непусто. Для combined-источника `13.89/13.98` берём anno `13.89`.

## Формат прототипа (по `part2_content_schema.md`)

Каждый прототип несёт: `id`, `cid` (авторский Б13-NNN — трассировка/имя SVG), `part:2`, `class`,
`method`, `uses[]`, `max_primary:2`, `stem` (а+б, LaTeX `\(…\)`), `answer_spec:{type:"manual"}`,
`answer:{general[],roots[]}`, `solution:{steps[],gen_groups[],below[],figure}`. Пример Б13-001
(`13.trig.homog.5.1`) совпал с иллюстративным примером схемы.

Замечания (для W13.1, не блокеры):
- **id-схема** провизорная: `13.<class-slug>.<method-slug>.<src>.<n>` (slug: trig/exp/log ·
  factor/quad/group/homog/other; лог/показ без method-сегмента). Точную id-схему фиксирует W13.1.
- **`figure`** — относительный путь `img/part2/<cid>.svg` (как ляжет в `content`); физически SVG лежат
  в `reports/part2_content_draft/img/part2/` до согласования места в W13.1.
- LaTeX из sympy использует `\frac` (схема в примере писала `\tfrac` — косметика, MathJax рендерит оба).
- Окружность — самодостаточный SVG (`xmlns`, `<rect>/<line>/<text>`, без script/MathJax) → безопасно
  через `<img>` (Решение 5 контракта).
